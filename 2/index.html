Attention based Spatial U-Net

Cleaning ultrasound images with deep learning

A deep learning approach to removing speckle noise from clinical ultrasound images while preserving the structures doctors actually need to see.

The Problem

Ultrasound imaging works by sending high-frequency sound waves into the body. A transducer on the skin picks up the echoes bouncing off organs, and a scanner reconstructs an image from the timing of those returns. It's real-time, non-invasive, and unlike X-rays, there's no ionizing radiation involved.

But there's a catch. When those sound waves bounce back, some arrive in phase and some arrive out of phase. This interference creates a grainy, speckled pattern across the image — speckle noise. It's not random static; it's baked into how ultrasound physics works. And it directly degrades the resolution and contrast that doctors rely on for diagnosis.

Traditional filters like mean and median can reduce some of this noise, but they blur edges and destroy fine details in the process. The image gets smoother, sure, but the structures that matter — organ boundaries, tissue textures, lesion margins — get smudged along with the noise.

What Existing Methods Do

Deep learning brought better results. A few notable approaches:

Multi-Level Wavelet CNN (MWCNN) breaks the image down into frequency sub-bands using wavelet transforms, processes each band with convolutional layers, then reconstructs the image. It's essentially a modified U-Net that works in the wavelet domain instead of the pixel domain. Decent results, but it treats all features equally regardless of importance.

DnCNN takes a different angle — instead of trying to predict the clean image directly, it learns the noise pattern itself. Subtract the predicted noise from the input and you get the clean image. Simple idea, works well for Gaussian noise, but struggles with speckle because speckle is multiplicative, not additive.

RIDNet introduced the idea of feature attention — a channel attention module that weighs which feature maps matter more. But it only looks at channel importance. It doesn't consider where in the image each feature is spatially important.

PRIDNet added multi-scale pyramid pooling and a kernel selection stage on top of channel attention. Better, but still treats features from different spatial locations the same way.

The common thread: none of these architectures pay attention to both what features matter (channel) and where they matter (spatial) at the same time.

The Proposed Architecture

The method has three stages, each solving a different part of the problem:

Input Image → Channel + Spatial Attention → SA-U-Net (Denoising) → Kernel Selection → Clean Output

Overall pipeline: attention first, then denoise, then adaptively select the right filter sizes

Stage 1: Dual Attention

Channel attention answers: which feature maps are most useful for denoising? It squeezes all the spatial information down using global average pooling, runs it through a small network, and produces a weight for each channel. Important features get amplified, irrelevant ones get suppressed.

Spatial attention answers: where in the image should the network focus? It takes the feature maps, applies max-pooling and average-pooling across channels to create a 2D importance map, then runs a convolution to produce a mask. Regions with meaningful structure get highlighted; flat noisy areas get de-emphasized.

Combining both means the network knows which features matter and where they matter. Channel attention picks the right channels; spatial attention tells it where to look within those channels. The two are added together (not multiplied) to keep gradients flowing smoothly during training.

Stage 2: Spatial Attention U-Net

The core denoising happens in a U-Net — an encoder-decoder architecture shaped like the letter U. The encoder compresses the image through successive convolution and pooling layers, extracting increasingly abstract features. The bottleneck at the bottom holds only the most essential representation. The decoder then upsamples back to full resolution.

Standard U-Nets use skip connections — direct links from each encoder level to the corresponding decoder level — to preserve spatial detail that would otherwise be lost during compression. But here's the problem: those skip connections also carry noise from the encoder straight into the decoder.

The fix: spatial attention gates at every skip connection. Before the encoder's features reach the decoder, they pass through a spatial attention filter that suppresses irrelevant and noisy information while keeping the structural details intact. The decoder gets clean spatial context instead of a mix of signal and noise.

This SA-U-Net runs five times in parallel with different kernel sizes, creating a pyramid structure. Each branch captures noise at a different scale, and the results are concatenated at the end.

Stage 3: Kernel Selection

Different regions of an image need different levels of filtering. Fine textures need small, precise kernels. Large smooth areas can use bigger kernels for stronger denoising. Rather than picking one kernel size for everything, this stage lets the network choose automatically.

Three parallel convolutions with different kernel sizes process the features. A global average pooling followed by two fully connected layers produces selection weights. The network learns which kernel size works best for each channel, adapting its receptive field on the fly. The process is called Selective Kernel convolution — split the input, fuse information across branches, then select the best combination.

Training

The model was trained on clinical ultrasound images resized to 512×512 pixels, with artificial speckle noise added at five variance levels: 0.01, 0.07, 0.1, 0.4, and 0.9. Data augmentation included 90°/180°/270° rotations and horizontal flips. The optimizer was Adam, and the loss function was mean squared error between the denoised output and the original clean image. Training took about 6 hours across 500 epochs.

Results

The method was compared against RIDNET and MWCNN across all five noise levels, measured by PSNR (higher is better — measures signal quality), SSIM (higher is better — measures structural similarity to the original), and visual inspection.

Method PSNR (avg) SSIM (avg) Verdict
RIDNET 22.4 ~0.78 Blurred structures, residual speckle at high noise
MWCNN 23.0 ~0.83 Better, but lost structural detail
Proposed (SA-U-Net) 29.4 ~0.85 Best denoising while preserving edges

At low noise (0.01), the proposed method hit a PSNR of 38.18 and SSIM of 0.961. Even at the highest noise level (0.9), it maintained recognisable structures where the other methods produced visibly degraded outputs. The key difference is edge preservation — the spatial attention gates in the skip connections prevent noise from leaking back into the reconstructed image while keeping organ boundaries and tissue boundaries sharp.

Why It Works

The insight is simple: not all features are equally important, and not all regions of the image need the same treatment. By asking "which channels matter?" and "where in the image should we focus?" before denoising, the network avoids the blind, uniform processing that makes other methods either leave noise behind or blur structures away.

The spatial attention on skip connections is the key contribution. Standard U-Nets blindly forward everything from encoder to decoder. Adding a learned gate that filters what gets through means the decoder works with cleaner context, producing sharper outputs without the noise that typically rides along skip connections.

For clinical ultrasound, where the difference between a clear organ boundary and a blurred one can affect diagnosis, this matters.
