What Actually Changed

When I worked on the CNN project in 2021, the way a neural network processed data was through small filters — a tiny window that slides across an image, pixel by pixel, picking up edges, textures, curves. Stack enough filters and the network starts recognizing things: this set of edges is an eye, that texture is fur, combine them and you get a cat. But each filter only sees its own small area. It doesn't know what's happening far away in the image. That's the core limitation — locality. A CNN is strong, but it's looking through a keyhole.

A CNN filter sliding across an image — it only sees a small patch at a time

The weights I learned about — those numbers the network keeps adjusting during training to get better — all lived inside these local filters. Backpropagation pushed errors backward through layers, nudging weights until they became useful. Attention, the concept I vaguely touched during the project, was the piece I didn't fully get at the time.

Let me explain attention from what I already knew:

In a CNN, if you're processing pixel 42, you only know about a few pixels around it — say 38 to 46, depending on filter size. Pixel 200? You have no clue what's there. But what if pixel 200 is the most important context for understanding pixel 42? In a CNN, that information has to survive through dozens of layers, slowly passing forward, getting weaker with each step.

Attention just removes that problem. Instead of a small sliding window, attention computes a relevance score between every element and every other element — all at once. Pixel 42 directly checks with pixel 200: "are you relevant to me?" and gets a weighted answer back. No waiting. No distance problem. Every part of the input can directly talk to every other part.

Self-attention — every word attends to every other word at once

How it works is through three things — Query, Key, and Value. Simple way to think about it: every element generates a question (Query), a tag describing itself (Key), and the actual data it carries (Value). To find out how relevant two elements are to each other, you match one's Query against another's Key. Good match means the Value gets high importance. Do this for every pair in the input and you get a full map of what matters to what.

Query, Key, Value — the three projections that make attention work

That's the difference. The weights in my CNN were local filter weights — they learned what a vertical edge looks like or what a texture boundary is. The weights in attention are relationship weights — they learn what's relevant to what across the entire input. Completely different type of learning.

Back then, I had the building blocks — neurons, weights, backpropagation, layers. What I didn't have was this one idea running at scale: let every part of the input talk to every other part, and let the network figure out which conversations matter. That's it. That single change is what turned narrow deep learning models into what everyone now calls AI.

The CNN I worked on could look at an image and classify it. The same core math — neurons, weights, activation functions, gradient descent — scaled up with attention, can now read code, hold conversations, and run commands in my terminal. Same ingredients. Different recipe. The recipe was attention.

What is a TUI?

So the daily driver of work for all of us includes interacting with UI to easily identify and use applications but at the system side it's just code and commands, all user interaction triggers code. A TUI means you effectively stand in between the user interface and the code getting triggered.

So the question would be why does it matter?

An effective TUI means the whole system can be controlled within the terminal app including all access to applications and workflows happening in the system. Sounds awesome right? A terminal doing all the work. Again the social media exploded with Claude Code's capability and I had all the people bragging about it muted in Twitter.

Two months later in April I finally heard and tried Claude Code. It was a great experience and my daily driver now includes Claude Code and it effortlessly does almost 40% of my work.

If I had to explain vaguely about its measuring system, I would put it this way. All AI models measure the work done for you through tokens. The more tokens you use per session the more number of neurons created and assigned weights, it also includes giving attention to the right set of neurons to get back correct results without hallucinating much. At the end it's all about how neurons are optimised and used in a manner that defines the performance of a model? That's a topic to explore and for another blog another time.
